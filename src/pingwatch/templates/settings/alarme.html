Aussetzer-Erkennung

Wann zaehlt ein Ping als verloren, ein Latenz-Ausreisser als Spike, ein Ziel als instabil ("FLAKY"). Defaults sind konservativ und seit Wochen produktiv.

N konsekutive Losses (Start) So viele Pings hintereinander muessen verloren gehen damit ein Outage beginnt. Schuetzt vor false positives durch einzelne Glitches.
K konsekutive OKs (Ende) So viele Pings hintereinander muessen wieder ankommen damit der Outage als beendet gilt. Verhindert Flapping bei wackliger Verbindung.
Spike-Threshold (ms) Ein Ping gilt als "Spike" wenn die RTT diesen Wert ueberschreitet. Default 80 ms.

Spike p50-Multiplikator ALTERNATIV — Spike wenn RTT > p50 * Multiplikator. Der hoehere der zwei Werte gewinnt. Verhindert dass auf einer schon-langsamen Strecke kein Spike mehr triggert.
MULTI-Fenster (ms) Wenn mehrere Ziele innerhalb dieses Fensters ausfallen, wird das als ein "MULTI"-Outage gruppiert statt N Einzel-Outages.
FLAKY-Threshold (%) Ein Ziel wird als "FLAKY" markiert wenn loss > X % ueber 24 h. Beeinflusst nur die Anzeige (Badge), nicht die Outage-Erkennung.
Toast bei Outage Kurz-Benachrichtigung rechts oben sobald ein Outage detected wurde.
Beep bei Outage Akustisches Signal via Browser-Audio-API (nur wenn der Browser Audio darf).
